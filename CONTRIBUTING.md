# コントリビューションガイド

このリポジトリへの貢献方法について説明します。

## 記事の投稿フロー

### 1. ブランチの作成

新しい記事を作成する際は、専用のブランチを作成してください。

```bash
git checkout -b feature/new-article-about-xxx
```

### 2. 記事の作成

`articles/` ディレクトリに新しいMarkdownファイルを作成します。

```bash
touch articles/your-article-title.md
```

### 3. 記事の執筆

Qiita CLIのフォーマットに従って記事を執筆してください。

```markdown
---
title: 記事のタイトル
tags:
  - タグ1
  - タグ2
private: false
---

# 記事の内容

ここに本文を書きます。
```

### 4. プルリクエストの作成

変更をコミットしてプルリクエストを作成します。

```bash
git add articles/
git commit -m "記事を追加: タイトル"
git push origin feature/new-article-about-xxx
```

GitHub上でプルリクエストを作成してください。

## CIチェック

プルリクエストを作成すると、以下のチェックが自動的に実行されます：

### 1. リンク切れチェック

- 記事内のすべてのリンクが有効かどうかをチェックします
- エラーが出た場合は、リンクを修正してください

### 2. Markdown Lint

- Markdownの文法やスタイルをチェックします
- reviewdogが問題点をコメントで指摘します
- 指摘された問題を修正してください

## コーディング規約

### Markdown

- 行の長さは120文字以内（コードブロック、テーブルは除く）
- 見出しレベルを適切に使用する
- コードブロックには言語を指定する

### コミットメッセージ

- 日本語で記述する
- 簡潔で分かりやすい内容にする

例：
```
記事を追加: GitHub Actionsの使い方
記事を更新: CI/CDの説明を追加
リンク切れを修正
```

## レビュー

プルリクエストは以下の点を確認してレビューされます：

- [ ] 記事の内容が適切か
- [ ] リンク切れがないか
- [ ] Markdown lintをパスしているか
- [ ] 不適切な内容が含まれていないか

## マージ

レビューが承認され、すべてのCIチェックをパスしたら、`main`ブランチにマージされます。
マージ後、自動的にQiitaに記事がアップロードされます。
